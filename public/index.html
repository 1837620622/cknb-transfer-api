2核心特性

  • 高可用容错：服务端内置多路容错机制，偶发网络波动或节点异常时自动切换并重试，流式输出开始前对用户完全无感知，保障稳定体验。
  • Claude 原生能力：/v1/messages 完整保留 tool_use、thinking、真实 usage、流式结构，适合 Claude Code 等原生客户端对接。
  • OpenAI 工具调用：/v1/chat/completions 带 tools 且目标为 Claude 时，自动转换到 Anthropic 协议，返回标准 tool_calls + finish_reason: tool_calls。
  • 双协议兼容：同一地址同时提供 OpenAI 与 Anthropic 两套接口，客户端可任选。
  • 多模型聚合：聚合 Claude、GPT、Gemini、DeepSeek、Llama 等多家顶级模型，一个端点统一调用。
  • 身份定制：模型对外自称 cknb-claude，适合作为自有品牌对外提供服务。
  • 推理增强：内置分步推理与自我验证引导，提升复杂问题的回答质量。

3对接 Claude Code

Claude Code 使用 Anthropic 协议，Base URL 指向本服务即可。

macOS / Linux

在终端执行，或写入 ~/.zshrc / ~/.bashrc：

export ANTHROPIC_BASE_URL="-"
export ANTHROPIC_AUTH_TOKEN="any-key"
export ANTHROPIC_API_KEY="any-key"
export ANTHROPIC_MODEL="gateway-claude-opus-4-8"
export ANTHROPIC_SMALL_FAST_MODEL="gateway-claude-opus-4-8"
claude

Windows PowerShell

$env:ANTHROPIC_BASE_URL = "-"
$env:ANTHROPIC_AUTH_TOKEN = "any-key"
$env:ANTHROPIC_API_KEY = "any-key"
$env:ANTHROPIC_MODEL = "gateway-claude-opus-4-8"
claude

settings.json 配置文件

编辑 ~/.claude/settings.json：

{
  "env": {
    "ANTHROPIC_BASE_URL": "-",
    "ANTHROPIC_AUTH_TOKEN": "any-key",
    "ANTHROPIC_API_KEY": "any-key",
    "ANTHROPIC_MODEL": "gateway-claude-opus-4-8"
  }
}
说明：本服务未设置客户端密钥校验，ANTHROPIC_AUTH_TOKEN 可填任意非空字符串。服务端已自动管理鉴权。

验证

curl -/v1/messages \\
  -H "Content-Type: application/json" \\
  -H "x-api-key: any-key" \\
  -d '{"model":"gateway-claude-opus-4-8","max_tokens":100,"messages":[{"role":"user","content":"你好"}]}'

6Tool Call 示例

Anthropic 协议

curl -/v1/messages \\
  -H "Content-Type: application/json" \\
  -d '{
    "model":"gateway-claude-opus-4-8",
    "max_tokens":1024,
    "messages":[{"role":"user","content":"巴黎天气如何？"}],
    "tools":[{"name":"get_weather","description":"获取城市天气","input_schema":{"type":"object","properties":{"city":{"type":"string"}},"required":["city"]}}]
  }'

OpenAI 协议（自动转换）

curl -/chat/completions \\
  -H "Content-Type: application/json" \\
  -d '{
    "model":"gateway-claude-opus-4-8",
    "max_tokens":1024,
    "messages":[{"role":"user","content":"巴黎天气如何？"}],
    "tools":[{"type":"function","function":{"name":"get_weather","description":"获取天气","parameters":{"type":"object","properties":{"city":{"type":"string"}},"required":["city"]}}}]
  }'

8常见问题

报 502 / 请求失败怎么办？

服务端已内置多路容错与自动重试。偶发错误会自动切换重试，用户通常无感知。若持续报错，稍后重试即可。

需要填真实 API Key 吗？

不需要。客户端 Authorization / x-api-key 填任意非空字符串即可，服务端已自动管理鉴权。如需限制访问，可在服务端设置 WORKER_API_KEY 环境变量。

有速率限制吗？

服务端采用多路并发容错架构，默认满足高并发需求。如需更大并发，可调整服务端配置。

支持流式吗？

支持。Anthropic stream:true 返回标准 SSE（message_start/content_block_delta/message_stop）；OpenAI stream:true 返回 chat.completion.chunk。流式开始前若出错会自动重试，开始输出后实时转发。

MCP 工具怎么用？

MCP server 在客户端（Claude Code / IDE）本地运行，本服务只提供模型 API 端点。配置好 Base URL 后，MCP 工具调用由客户端发起，模型通过 tool_use 返回调用指令。

模型自称什么？

模型对外自称 cknb-claude（由 CKNB 开发），适合作为自有品牌对外提供服务。问"你是什么模型"会回答"我是 cknb-claude 模型"。
